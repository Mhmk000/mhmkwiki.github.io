MACRO-TOPIC: Ethical Hacking & Security (15)

Overview

This macrotopic covers the principles of ethical hacking, common techniques used in security testing, and practical measures to protect systems and data. The content below includes learning objectives, a short hands-on checklist, recommended (safe) tools to explore, and pointers for further study.

Resources and activities

You can add the supplied resource file to the site folder so students can consult it locally, or link to external material from the sections below. Keep all hands-on work inside an isolated lab environment and always obtain explicit permission before testing any real systems.

Quick checklist for exercises

  • Set up an isolated test network or virtual machines.
  • Verify that you have permission to perform any tests.
  • Collect passive information (open sources, public records).
  • Perform non-intrusive scans and document results carefully.
  • Report findings and propose mitigation steps instead of exploiting vulnerabilities.

Suggested hands-on tasks

  • Map a small lab network and identify exposed services.
  • Practice scanning for common weaknesses using safe settings.
  • Create a short security report with recommendations.

Recommended reading & references

  • OWASP Top Ten — common web application risks.
  • Introductory guides on responsible disclosure and legal/ethical frameworks.

Suggested topics

  • Ethical hacking methodology (reconnaissance, scanning, exploitation, reporting)
  • Common vulnerabilities and mitigation (OWASP Top 10)
  • Basic pentesting tools and safe usage
  • Legal and ethical considerations
